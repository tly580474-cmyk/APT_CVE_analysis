# APT攻击情报分析项目计划文档

## 1. 项目概述

### 1.1 项目主旨
本项目旨在构建一个APT（高级持续性威胁）攻击情报分析平台，通过可视化展示和AI分析，帮助用户更好地理解和应对网络安全威胁。

### 1.2 核心功能
- **本地文档数据展示**：将本地的APT攻击情报文档数据展示在网页上
- **CVE漏洞信息集成**本地cve_repo
- **威胁等级可视化**：将文档中的漏洞威胁等级进行可视化展示
- **热度排行可视化**：展示威胁热度排行
- **词云生成**：基于文档内容生成词云
- **雷达图生成**：生成威胁分析雷达图
- **用户上传与AI分析**：允许用户上传文档，使用AI分析文档主要内容
- **论坛系统**：用户登录后可进行交流
- **用户登录系统**：简单的用户认证机制

## 3. 项目架构设计

### 3.1 整体架构
采用前后端分离架构：
- **前端**：React单页应用，负责用户界面和数据展示
- **后端**：Express API服务，负责数据处理、存储和AI集成
- **数据库**：PostgreSQL (使用Docker容器化)，存储用户数据、文档信息和论坛内容

### 3.2 模块划分

#### 前端模块
1. **认证模块**：用户登录、注册、密码重置
2. **文档管理模块**：文档上传、列表展示、详情查看
3. **CVE模块**：CVE漏洞信息展示、搜索、详情查看
4. **可视化模块**：威胁等级图表、热度排行、词云、雷达图
5. **论坛模块**：帖子列表、发布、评论、回复
6. **用户模块**：个人信息管理

#### 后端模块
1. **认证API**：用户注册、登录、认证
2. **文档API**：文档上传、管理、分析
3. **CVE API**：CVE漏洞信息获取、搜索、管理
4. **AI分析API**：调用OpenAI API进行文档分析
5. **论坛API**：帖子、评论管理
6. **数据统计API**：威胁等级、热度数据统计

## 6. 后端开发方案

### 6.1 API设计

#### 认证API
- `POST /api/auth/register`：用户注册
- `POST /api/auth/login`：用户登录
- `GET /api/auth/me`：获取当前用户信息

#### 文档API
- `POST /api/documents`：上传文档
- `GET /api/documents`：获取文档列表
- `GET /api/documents/:id`：获取文档详情
- `PUT /api/documents/:id`：更新文档
- `DELETE /api/documents/:id`：删除文档

#### CVE API
- `GET /api/cve`：获取CVE列表
- `GET /api/cve/:id`：获取CVE详情
- `GET /api/cve/search`：搜索CVE
- `GET /api/cve/hot`：获取热门CVE

#### AI分析API
- `POST /api/ai/analyze`：分析文档内容

#### 论坛API
- `POST /api/forum/posts`：发布帖子
- `GET /api/forum/posts`：获取帖子列表
- `GET /api/forum/posts/:id`：获取帖子详情
- `POST /api/forum/posts/:id/comments`：添加评论

#### 数据统计API
- `GET /api/stats/threat-levels`：获取威胁等级统计
- `GET /api/stats/heat-rankings`：获取热度排行
