## 4 前端API服务

### 4.1 API服务结构

前端通过 `services/api.js` 封装API调用，主要包括：

- authAPI：认证相关接口
- documentAPI：文档相关接口
- cveAPI：CVE相关接口
- forumAPI：论坛相关接口
- aiAnalysisAPI：AI分析相关接口
- statsAPI：统计相关接口

### 4.2 主要API调用

| 服务 | 方法 | 功能 |
|------|------|------|
| authAPI.login | POST | 用户登录 |
| authAPI.register | POST | 用户注册 |
| authAPI.getProfile | GET | 获取用户资料 |
| documentAPI.getDocuments | GET | 获取文档列表 |
| documentAPI.uploadDocument | POST | 上传文档 |
| cveAPI.getCVEs | GET | 获取CVE列表 |
| cveAPI.searchCVEs | GET | 搜索CVE |
| forumAPI.getPosts | GET | 获取帖子列表 |
| forumAPI.createPost | POST | 创建帖子 |
| aiAnalysisAPI.analyzeText | POST | 分析文本 |
| statsAPI.getCVEAnalysis | GET | 获取CVE分析数据 |

## 9 安全考虑

1. **JWT认证**：使用JWT进行用户认证，确保API安全
2. **密码加密**：用户密码使用bcrypt加密存储
3. **权限控制**：基于角色的访问控制，区分普通用户和管理员
4. **文件上传安全**：限制文件类型和大小，防止恶意文件上传
5. **输入验证**：对所有用户输入进行验证，防止注入攻击
6. **CORS配置**：合理配置CORS，限制跨域请求

## 11 总结

本项目采用前后端分离架构，前端使用React + Ant Design，后端使用Node.js + Express + PostgreSQL。系统提供了完整的APT攻击情报分析功能，包括文档管理、CVE漏洞信息、AI分析、论坛交流等模块。通过合理的路由设计和API接口规划，实现了功能的模块化和可扩展性。

系统的主要特点：
- 完整的用户认证和授权系统
- 支持文档上传和管理
- 集成CVE漏洞信息
- 提供数据可视化分析
- 支持AI智能分析
- 实现论坛交流功能
- 安全可靠的API设计
- 易于部署和维护

该架构设计满足了APT攻击情报分析的核心需求，为安全分析人员提供了一个高效、直观的情报分析平台。
